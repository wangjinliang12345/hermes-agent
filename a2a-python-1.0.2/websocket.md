# A2A Client WebSocket Transport

## Overview

This change adds a **WebSocket-based transport** to the A2A Python client. Instead of sending A2A requests via HTTP JSON-RPC directly to an agent, the client now runs an embedded **WebSocket server** that accepts inbound connections from agent subs. Each sub is identified by its remote IP/port and assigned a unique `agent_id`. The client routes A2A method calls to the correct sub through its persistent WebSocket connection.

This is useful when:
- The agent (sub) is behind NAT/firewall and cannot expose an HTTP endpoint.
- You want a persistent bidirectional channel rather than per-request HTTP calls.
- You need the client to act as a gateway/proxy that manages many agent connections.

---

## Architecture

```
+------------------+          ws://              +------------------+
|   A2A Client     |  <---------------------->  |   Agent (sub)    |
|                  |       persistent conn       |                  |
|  WebSocketServer |                            |  WebSocket client|
|       ^          |                            |                  |
|       | routes   |                            |                  |
|  WebSocketTransport                            |                  |
|       ^          |                            |                  |
|       | uses     |                            |                  |
|   ClientFactory  |                            |                  |
|   BaseClient     |                            |                  |
+------------------+                            +------------------+
```

### Key Components

1. **A2AWebSocketServer** (`src/a2a/client/websocket_server.py`)
   - Binds to a host/port and accepts WebSocket connections.
   - On connect, records `(remote_ip, remote_port)` and generates a UUID `agent_id`.
   - Maintains `agent_id -> WebSocketProtocol` mappings.
   - Provides `send_request(agent_id, method, payload)` for request/response.
   - Provides `send_stream_request(agent_id, method, payload)` for streaming responses.

2. **WebSocketTransport** (`src/a2a/client/transports/websocket.py`)
   - Implements `ClientTransport`.
   - Translates every A2A method (`send_message`, `get_task`, `cancel_task`, etc.) into a WebSocket message routed by `agent_id`.
   - Serializes/deserializes protobuf messages via `json_format`.

3. **ClientConfig** (`src/a2a/client/client.py`)
   - New field: `websocket_server: A2AWebSocketServer | None`
   - Injected into `ClientFactory` when building a `WEBSOCKET` transport.

4. **ClientFactory** (`src/a2a/client/client_factory.py`)
   - Registers a default producer for `TransportProtocol.WEBSOCKET`.
   - When creating a client with a `WEBSOCKET` interface, the factory instantiates `WebSocketTransport` and passes the configured `websocket_server`.

## Usage

### 1. Start the WebSocket Server

```python
import asyncio
from a2a.client.websocket_server import A2AWebSocketServer

async def main():
    server = A2AWebSocketServer(host='0.0.0.0', port=8765)
    await server.start()
    # subs will now connect to ws://<host>:8765
    # await server.stop()  # when shutting down

asyncio.run(main())
```

### 2. Create an Agent Card with WEBSOCKET Interface

The `url` field of the interface should be the `agent_id` assigned by the server when the sub connects.

```python
from a2a.types.a2a_pb2 import AgentCard, AgentInterface, AgentCapabilities
from a2a.utils.constants import TransportProtocol

card = AgentCard(
    name='RemoteAgent',
    supported_interfaces=[
        AgentInterface(
            protocol_binding=TransportProtocol.WEBSOCKET,
            url=agent_id,   # UUID generated by A2AWebSocketServer
        )
    ],
    capabilities=AgentCapabilities(streaming=False),
    # ... other required fields
)
```

### 3. Create a Client via ClientFactory

```python
from a2a.client.client_factory import ClientFactory
from a2a.client.client import ClientConfig
from a2a.utils.constants import TransportProtocol

config = ClientConfig(
    supported_protocol_bindings=[TransportProtocol.WEBSOCKET],
    websocket_server=server,
)

factory = ClientFactory(config)
client = factory.create(card)

# Now use the client normally
request = SendMessageRequest(...)
async for response in client.send_message(request):
    print(response)

await client.close()
```

## Message Protocol (Sub Side)

When an agent sub connects to the WebSocket server, it receives JSON messages in the following format:

### Request from Server

```json
{
  "request_id": "<uuid>",
  "method": "SendMessage",
  "payload": { /* protobuf message as dict */ }
}
```

Supported methods: `SendMessage`, `SendStreamingMessage`, `GetTask`, `ListTasks`, `CancelTask`, `CreateTaskPushNotificationConfig`, `GetTaskPushNotificationConfig`, `ListTaskPushNotificationConfigs`, `DeleteTaskPushNotificationConfig`, `SubscribeToTask`, `GetExtendedAgentCard`.

### Response from Sub (normal)

```json
{
  "request_id": "<uuid>",
  "payload": { /* protobuf response as dict */ }
}
```

### Response from Sub (streaming)

For streaming methods, the sub may send multiple chunks. The last chunk should include `stream_done: true`:

```json
{
  "request_id": "<uuid>",
  "payload": { /* partial response dict */ },
  "stream_done": true
}
```

If `stream_done` is omitted, the client transport assumes the stream continues until timeout.

---

## Agent ID Assignment

The server maps each WebSocket connection by its remote address:

```
agent_id = uuid4()
agent_ids[(remote_ip, remote_port)] = agent_id
connections[agent_id] = websocket_protocol
```

If the same `(ip, port)` reconnects after a disconnect, a **new** `agent_id` is generated. The caller should query `server._connections.keys()` or expose a registry API to discover active agent IDs.

## 端到端演示（End-to-End Demo）

仓库根目录提供了一个可运行的演示脚本 `demo_websocket_client.py`，完整展示了 WebSocket 传输的工作流程：

1. 启动 `A2AWebSocketServer`，监听 `ws://127.0.0.1:18767`
2. 启动模拟的 Agent Sub（普通的 `websockets` 客户端），连接到 Server
3. 构建带有 `WEBSOCKET` 接口的 `AgentCard`（`url` 填写 server 分配的 `agent_id`）
4. 通过 `ClientFactory` + `WebSocketTransport` 创建 A2A `Client`
5. 分别发送**非流式**和**流式**消息进行测试

### 运行演示

```bash
cd a2a-python-1.0.2
PYTHONPATH=src python demo_websocket_client.py
```

### 预期输出

```text
[Server] Started on ws://127.0.0.1:18767
[Sub] Connected to ws://127.0.0.1:18767
[Client] Discovered agent_id: <uuid>
[Client] Created A2A client with WebSocket transport

--- Non-streaming send_message ---
[Sub] Replied to SendMessage
Task ID : demo-task-sync
State   : 3
Reply   : Hello from WebSocket sub! You said: Hello WebSocket agent!

--- Streaming send_message ---
Chunk 1: state=2 | text=Streaming chunk 1
Chunk 2: state=2 | text=Streaming chunk 2
Chunk 3: state=2 | text=Streaming chunk 3
Chunk 4: state=3 | text=Stream finished

Total stream chunks received: 4
[Client] Closed
[Sub] Stopped
[Server] Stopped
```

### 演示中的关键要点

- **Agent ID 发现**：Sub 连接后，Server 会自动分配一个 UUID 作为 `agent_id`。演示脚本通过 `server._connections.keys()` 读取已连接 Sub 的 ID。
- **非流式客户端**：将 `ClientConfig(streaming=False)`，`BaseClient.send_message()` 会调用 `transport.send_message()`，返回单个 `StreamResponse`。
- **流式客户端**：设置 `ClientConfig(streaming=True)`，并确保 `AgentCapabilities(streaming=True)`，`BaseClient.send_message()` 会调用 `transport.send_message_streaming()`，持续 yield 多个 chunk，直到收到 `stream_done=True`。
- **消息协议**：Mock Sub 收到的请求格式为 `{"request_id": "...", "method": "SendMessage", "payload": {...}}`，回复格式为 `{"request_id": "...", "payload": {"task": {...}}}`. 流式传输时，最后一个 chunk 必须包含 `"stream_done": true`。

### 快速代码片段

如果你想在自己的代码中使用 WebSocket 传输，可以参考以下最小示例：

```python
from a2a.client.websocket_server import A2AWebSocketServer
from a2a.client.client_factory import ClientFactory
from a2a.client.client import ClientConfig
from a2a.types.a2a_pb2 import AgentCard, AgentInterface, AgentCapabilities, SendMessageRequest
from a2a.utils.constants import TransportProtocol

# 1. 启动 WebSocket Server
server = A2AWebSocketServer(host='127.0.0.1', port=8765)
await server.start()

# 2. 等待 Sub 连接后，获取其 agent_id
agent_id = list(server._connections.keys())[0]

# 3. 构建带有 WEBSOCKET 接口的 AgentCard
card = AgentCard(
    name='MyAgent',
    capabilities=AgentCapabilities(streaming=True),
    supported_interfaces=[
        AgentInterface(
            protocol_binding=TransportProtocol.WEBSOCKET,
            url=agent_id,
        )
    ],
    # ... 填写其他必填字段
)

# 4. 创建 Client
config = ClientConfig(
    streaming=True,
    supported_protocol_bindings=[TransportProtocol.WEBSOCKET],
    websocket_server=server,
)
factory = ClientFactory(config)
client = factory.create(card)

# 5. 发送消息
request = SendMessageRequest()
request.message.role = 'ROLE_USER'
request.message.parts.add().text = 'Hello'

async for resp in client.send_message(request):
    print(resp.task.status.message.parts[0].text)
```
